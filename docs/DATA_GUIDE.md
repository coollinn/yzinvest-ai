# YZInvest AI — 数据策略与更新指南

> 本文档记录数据来源、API参数发现过程、分层更新策略，防止遗忘。

---

## 一、数据来源

**东方财富（EastMoney）免费 API**，稳定、无需认证，是主力数据源。

主要接口：
- **行情列表**：`push2delay.eastmoney.com/api/qt/clist/get` — 股票基础信息 + 实时价格
- **K线数据**：`push2his.eastmoney.com/api/qt/stock/kline/get` — 历史K线
- **财务数据**：`datacenter.eastmoney.com` — 财务报表、指标
- **公告数据**：`np-anotice-stock.eastmoney.com` — 公司公告
- **新闻数据**：`finance.eastmoney.com/a/` — 财经新闻

## 二、FS 参数发现过程

东方财富 API 的 `fs` 参数决定板块范围，格式：`m:{market}+t:{type}`。

### 2.1 参数格式说明

| 代码 | 含义 |
|:---|:---|
| `m:0` | 上交所 |
| `m:1` | 深交所 |
| `t:6` | 沪市主板 |
| `t:13` | 深市主板（早期） |
| `t:2` | 深市主板 |
| `t:23` | 科创板（深市部分） |
| `t:80` | 科创板（上交所）、创业板（深交所） |
| `t:81` | 科创板（补充） |
| `t:82` | 北交所 |

> **注意**：`t:80` 在沪市 = 科创板，在深市 = 创业板，需配合 `m:0/m:1` 区分。

### 2.2 各板块测试结果

| 板块参数名 | FS 参数 | 数据量 |
|:---|:---|---:|
| 沪深主板（旧参数） | `m:0+t:6,m:0+t:13,m:1+t:2,m:1+t:23` | ~4090 条 |
| 科创板 | `m:0+t:80,m:0+t:81,m:1+t:23,m:1+t:80` | ~8907 条 |
| 创业板 | `m:0+t:80,m:1+t:80` | ~1438 条 |
| 北交所 | `m:0+t:82,m:1+t:82` | 网络超时（少量） |
| **沪深全量（推荐）** | `m:0+t:6,m:0+t:13,m:0+t:80,m:0+t:81,m:1+t:2,m:1+t:23,m:1+t:80,m:0+t:82,m:1+t:82` | **~12386 条** |

**测试脚本**：`scripts/test_fS_params.py`

```bash
cd /Users/leon.chen/work/ai-dev-projects/yzinvest-ai
python3 scripts/test_fS_params.py
```

### 2.3 关键发现

1. **旧参数** `m:0+t:6,m:0+t:13,m:1+t:2,m:1+t:23` 只覆盖约 4090 条，漏掉了大量科创板和创业板股票
2. **`total` 字段有时返回字符串而非整数**，导致脚本中 `page * page_size >= total` 比较报错
3. **API 每页最多返回 100 条**（`pz=200` 参数被忽略）
4. **全量数据需要约 124 页**（100条/页）

## 三、数据分层架构

```
┌─────────────────────────────────────────────┐
│  Layer 1: 股票基础信息 (stocks)              │  ← 变更极少，手动更新
│  ts_code / symbol / name / exchange          │
├─────────────────────────────────────────────┤
│  Layer 2: 行情数据 (stock_daily)             │  ← 交易日每日增量
│  trade_date / open / high / low / close     │
├─────────────────────────────────────────────┤
│  Layer 3: 财务数据 (stock_financial)         │  ← 季报/年报定期更新
│  营收 / 利润 / 资产 / 负债                   │
├─────────────────────────────────────────────┤
│  Layer 4: 分析数据 (stock_valuation/notes)   │  ← 按需计算/用户生成
│  估值指标 / 自选股 / 个人笔记                 │
└─────────────────────────────────────────────┘
```

### 3.1 Layer 1 — 股票基础信息（stocks 表）

**何时更新**：
- 新股上市时（每年约 300-500 家）
- 退市/暂停上市时
- 公司更名时
- **建议频率**：每季度手动更新一次，或发现数据缺失时触发

**当前状态**：5975 条（远程D1）

**更新方法**：
```bash
# 方法1：使用管理脚本
cd /Users/leon.chen/work/ai-dev-projects/yzinvest-ai
./scripts/manage.sh stock-sync

# 方法2：直接运行获取脚本（全量 12386 条）

# 方法3：分模块获取（节省时间）
python3 scripts/sync_stocks_sql.py 100 1 "m:0+t:6,m:0+t:13,m:1+t:2,m:1+t:23"  > /tmp/sh_main.sql
python3 scripts/sync_stocks_sql.py 100 1 "m:0+t:80,m:0+t:81,m:1+t:23,m:1+t:80" > /tmp/kechuang.sql
# 合并后导入
```

### 3.2 Layer 2 — 行情数据（stock_daily 表）

**何时更新**：
- 每个交易日结束后（约 16:00-17:00）
- **建议频率**：每日收盘后增量更新（只更新当日有交易的股票）

**当前状态**：3244 条（远程D1）

**更新方法**：
```bash

./scripts/manage.sh daily-sync 20260501

# 或通过东方财富实时接口逐股票拉取
# 脚本：scripts/sync_daily_prices.py（需实现）
```

> **注意**：行情数据量大，全量历史K线需另外拉取 `push2his` 接口。

### 3.3 Layer 3 — 财务数据（stock_financial 表）

**何时更新**：
- 财报发布季（4月、8月、10月高峰期）
- **建议频率**：每季度财报发布后增量更新

**数据源**：`datacenter.eastmoney.com/seon/api/data/api`（财务报表）
**脚本**：`scripts/sync_financial.py`（需实现）

### 3.4 Layer 4 — 分析数据

- **stock_valuation**：估值指标（PE、PB、PS 等），由实时价格 + 财务数据计算
- **favorites**：用户自选股，用户自行管理
- **notes**：用户笔记，用户自行管理

## 五、D1 数据库操作注意事项

1. **不支持事务语句**：`BEGIN TRANSACTION` / `COMMIT` 需移除
2. **每批限制**：建议每次不超过 100 条 INSERT（单条语句），大文件分批导入
3. **本地数据库文件锁**：运行 `wrangler dev` 时本地 D1 被锁定，无法同时执行 `wrangler d1 execute --local`
4. **导出 SQL 后导入前**：删除文件中的 `BEGIN TRANSACTION` 和 `COMMIT`

## 六、东方财富 API 关键字段

### 行情列表字段（push2delay）

| 字段 | 说明 | 注意 |
|:---|:---|:---|
| `f12` | 股票代码 | 如 `000001` |
| `f14` | 股票名称 | |
| `f3` | 涨跌幅 % | 单位：%，需除以 100 |
| `f4` | 当前价格 | 单位：分，需除以 100 |
| `f5` | 开盘价 | 单位：分 |
| `f6` | 最高价 | 单位：分 |
| `f7` | 最低价 | 单位：分 |
| `f15` | 今日最高（原始） | 单位：分 |
| `f16` | 今日最低（原始） | 单位：分 |
| `f17` | 今开价格 | 单位：分 |
| `f18` | 成交量 | 手 |
| `f20` | 成交额 | 元 |

**ts_code 转换规则**：
- `f12` 以 `6` 开头 → 上交所（SH）
- `f12` 以 `0`/`3` 开头 → 深交所（SZ）
- `f12` 以 `4`/`8`/`9` 开头 → 北交所（BJS → BJ）
- 格式：`{f12}.{exchange}`
